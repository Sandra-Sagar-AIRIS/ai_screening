Circular Dependencies — Screening Service

One indirect coupling found: Screening → Auth → (Monolith).

routes/ai_screening.py imports get_current_user and require_permission from app.core.dependencies.
core/dependencies.py queries Profile (auth table) on every request.

When Screening Service is extracted: it needs JWT validation. It will call GET {AUTH_URL}/internal/users/{id} to validate tokens. This means:
Screening Service → Auth Service (for all JWT validation) AND Screening Service → Monolith (for candidate/job/pipeline data).

This is a fan-out pattern, not a circular dependency. Auth Service does not call Screening Service. Monolith does not call Auth Service in the request path (only during signup). The dependency graph is acyclic.
